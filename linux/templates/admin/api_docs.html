REST API reference for programmatic access to this news portal.

Authentication

Every API endpoint (except GET /api/version and GET /api/search) requires authentication. Two methods are accepted:

  • Session cookie — automatically used when you are logged in to the admin panel.
  • API key header — send {{if .APIKey}}X-API-Key: {{.APIKey}}{{else}}X-API-Key: <your-key>{{end}} in the request. The key is set via the API_KEY environment variable.
{{if eq .APIKey ""}}
Az API_KEY környezeti változó ezen az oldalon nincs beállítva. Az API-t kulcsos hitelesítéssel nem lehet használni.
{{end}}

Unauthenticated requests receive:

HTTP/1.1 401 Unauthorized
Content-Type: application/json

{"error":"Hitelesítés szükséges"}

curl example (API key):

curl -H "X-API-Key: {{if .APIKey}}{{.APIKey}}{{else}}YOUR_KEY{{end}}" {{if .SiteURL}}{{.SiteURL}}{{else}}https://example.com{{end}}/api/articles

Site Statistics

MethodURLAuthDescription
GET /api/stats Required Site-wide statistics (same data as the dashboard)

Response:

{
  "total_articles":     142,
  "published_articles": 130,
  "draft_articles":     12,
  "total_categories":   8,
  "total_authors":      5,
  "views_today":        47,
  "views_7days":        312,
  "views_30days":       1240,
  "top_today":   { <article object or null> },
  "top_7days":   { <article object or null> },
  "top_30days":  { <article object or null> }
}

The top_today, top_7days, top_30days fields each contain the full article object (same shape as in List articles) with the most unique readers in the given period, or null if there is no data yet.

curl:

curl -H "X-API-Key: {{if .APIKey}}{{.APIKey}}{{else}}YOUR_KEY{{end}}" {{if .SiteURL}}{{.SiteURL}}{{else}}https://example.com{{end}}/api/stats

Articles

List articles
MethodURLAuth
GET /api/articles Required

Query parameters:

  • limit — number of results (default: 20, max: 100)
  • offset — zero-based skip (default: 0)
  • published — true to return only published articles; omit for all

Response:

{
  "total": 142,
  "limit": 20,
  "offset": 0,
  "articles": [
    {
      "id":           "664a1b2c3d4e5f6789abcdef",
      "title":        "Example Article",
      "slug":         "example-article",
      "summary":      "Short summary…",
      "content":      "Markdown content…",
      "author":       "Jane Doe",
      "author_id":    "664a000000000000000000aa",
      "category":     "Technology",
      "category_id":  "664a000000000000000000bb",
      "tags":         ["go", "api"],
      "image_url":    "{{if .SiteURL}}{{.SiteURL}}{{else}}https://example.com{{end}}/images/photo.jpg",
      "published":    true,
      "published_at": "2025-06-01T10:00:00Z",
      "created_at":   "2025-06-01T09:55:00Z",
      "updated_at":   "2025-06-01T10:00:00Z"
    }
  ]
}

curl:

curl -H "X-API-Key: {{if .APIKey}}{{.APIKey}}{{else}}YOUR_KEY{{end}}" \
  "{{if .SiteURL}}{{.SiteURL}}{{else}}https://example.com{{end}}/api/articles?limit=10&offset=0&published=true"
Get one article
MethodURLAuth
GET /api/articles/{id} Required

{id} is the MongoDB ObjectID (24-char hex string).

Response 200: article object (same shape as in list).
Response 404: {"error":"A cikk nem található"}

curl -H "X-API-Key: {{if .APIKey}}{{.APIKey}}{{else}}YOUR_KEY{{end}}" \
  {{if .SiteURL}}{{.SiteURL}}{{else}}https://example.com{{end}}/api/articles/664a1b2c3d4e5f6789abcdef
Create article
MethodURLAuth
POST /api/articles Required

Request body (JSON):

FieldTypeRequiredNotes
titlestringYesArticle title. Slug is auto-generated.
summarystringShort excerpt shown in listings.
contentstringMarkdown body.
authorstringDisplay name (used when author_id is absent).
author_idstringObjectID of an existing author.
categorystringDisplay name.
category_idstringObjectID of an existing category.
tagsstring[]Tag list.
image_urlstringFeatured image URL. If S3 is configured, external URLs are auto-imported to the media library.
publishedboolPublish immediately (default: false).
publish_atstring (RFC3339)Scheduled publish time. Null or omit to use the current time when published=true.

curl:

curl -X POST -H "X-API-Key: {{if .APIKey}}{{.APIKey}}{{else}}YOUR_KEY{{end}}" -H "Content-Type: application/json" \
  -d '{
    "title":      "My New Article",
    "summary":    "A short summary.",
    "content":    "## Introduction\n\nHello world.",
    "category":   "Technology",
    "category_id":"664a000000000000000000bb",
    "tags":       ["go", "tutorial"],
    "published":  true
  }' \
  {{if .SiteURL}}{{.SiteURL}}{{else}}https://example.com{{end}}/api/articles

Response 201: the created article object.
Response 400: {"error":"A cím megadása kötelező"}

Update article
MethodURLAuth
PUT /api/articles/{id} Required

Same body fields as Create article, plus:

FieldTypeNotes
update_slugboolWhen true the slug is regenerated from the title (default: false — existing URLs stay stable).
Omit update_slug (or set it to false) to keep the existing URL intact. Setting it to true will break any external links to the old URL.

curl:

curl -X PUT -H "X-API-Key: {{if .APIKey}}{{.APIKey}}{{else}}YOUR_KEY{{end}}" -H "Content-Type: application/json" \
  -d '{
    "title":     "Updated Title",
    "content":   "New content.",
    "published": true
  }' \
  {{if .SiteURL}}{{.SiteURL}}{{else}}https://example.com{{end}}/api/articles/664a1b2c3d4e5f6789abcdef

Response 200: updated article object.

Create category
MethodURLAuth
POST /api/categories Required
FieldTypeRequiredNotes
namestringYesSlug is auto-generated from name (Hungarian accents handled).
descriptionstring
iconstringBootstrap Icons name without the bi- prefix, e.g. cpu.

  {{if .SiteURL}}{{.SiteURL}}{{else}}https://example.com{{end}}/api/categories/664a000000000000000000bb
